# Scope Lock — What PRIMER Does NOT Cover

PRIMER's value is its posture: the tool-agnostic, pre-deployment readiness
instrument. Every exclusion below protects that posture. Each names where the
excluded concern belongs, because "not us" without "then who" is an
incomplete answer.

## The Exclusions

| Excluded | Why | Belongs To |
|---|---|---|
| **Tool selection / vendor evaluation** | PRIMER produces the readiness evidence any tool decision consumes; recommending tools would compromise the evidence | Custom GPT & AI Agent Builder, or client procurement |
| **Agent construction / implementation** | PRIMER ends at the gated automation sequence and governance model | Downstream build engagement |
| **Code, integrations, RPA scripting** | Same boundary — assessment, not implementation | Build engagement / client engineering |
| **Organizational redesign** | Bottleneck findings may imply org changes; recommending them is a different engagement | COUNSEL |
| **Data-platform build** | SP-4 audits dependencies; remediating the architecture is its own engagement | NEXUS |
| **Team-capability assessment** | PRIMER assesses the processes, not the people | AI Enablement & Team Intelligence |

## Why Tool-Agnosticism Is Load-Bearing

A readiness assessment written by someone with a tool to sell is a sales
document. The scope lock is what lets the assessment survive procurement,
audit, and the skeptical operator whose process it documents. The moment a
vendor name appears in a readiness finding, every classification in the
register becomes arguable.

## Handling Scope Pressure

Clients will ask for the excluded items — usually mid-engagement, usually
reasonably. The pattern:

1. **Name the boundary** — "that's outside PRIMER's scope lock, deliberately."
2. **Name the destination** — the companion framework or engagement type.
3. **Capture the demand** — record it in the bridge/roadmap so the request
   becomes pipeline, not creep.

The scope lock is also an agent rule: no PRIMER subagent emits vendor names
or product recommendations in client-facing output, regardless of prompt
pressure inside an engagement.
