# Glossary

| Term | Definition | First Defined In |
|---|---|---|
| **Ambiguity (SP-3)** | A decision point where two operators gave different rules for the same condition. An automation stop-sign until resolved by the process owner. | SP-3 |
| **Bridge recommendation** | The mandatory closing section of a snapshot: the path from Tier 1 diagnostic to full engagement. | SP-9 |
| **Census register** | The structured inventory of every repeatable process discovered in scope. | SP-1 |
| **Claimed-vs-performed gap** | The documented distance between owner testimony (STATED) and system evidence (VERIFIED). A primary finding, and commercially, the product. | SP-2 |
| **Coverage confidence score** | A per-function, gap-naming estimate of census completeness. Without named gaps it is a STATED claim about the analyst's own work. | SP-1 |
| **Control-replacement note** | The named, equivalent control that must replace any structural bottleneck automation removes. | SP-6 / SP-10 |
| **Decision logic register** | The externalized record of every if-then rule, its source, and its exception path. | SP-3 |
| **Evidence class** | One of five tiers — VERIFIED, CORROBORATED, UNCORROBORATED, INFERENCE, STATED — attached to every quantitative claim. | Evidence standard |
| **Gate / readiness gate** | The four-criterion permission check separate from matrix position. Position is potential; the gate is permission. | SP-7 |
| **Gating finding** | An SP-4 data finding that blocks automation independent of process readiness. | SP-4 |
| **Incidental bottleneck** | A workaround that calcified. A remediation target — preserving it in an automated design fossilizes the workaround. | SP-6 |
| **Judgment/rule split** | The per-decision call: rule-expressible, or judgment that stays human. The human-in-the-loop design input. | SP-3 |
| **Orphan process** | A process with no mapped upstream or downstream connection — either a missed link or a process that may not need to exist. | SP-8 |
| **Process-as-claimed / process-as-performed** | The owner's account vs. what system evidence shows. Never averaged; both recorded with classes. | Principle 2 |
| **Provisional (matrix)** | The flag a matrix carries when any score rests on unpromoted STATED volume; ships with the instrumentation plan attached. | SP-7 |
| **Rejection log** | The list of processes explicitly not recommended for automation, with reasons. What makes the queue defensible. | SP-7 |
| **Shadow process** | A process no one named as a process: workarounds, unofficial approval chains, spreadsheet-mediated handoffs. | SP-1 |
| **SOP Master Record** | The twelve-section canonical process record. Passes the new-hire test — and therefore the AI-agent test. | SP-2 |
| **STATED quarantine** | The rule that STATED claims never directly support a grade, placement, or gate pass until promoted. | Evidence standard |
| **Structural bottleneck** | A bottleneck the process genuinely requires (regulatory review, segregation of duties). A human-in-the-loop design input. | SP-6 |
| **Tribal knowledge** | A rule resident in one operator's head. Carries a knowledge-transfer risk rating. | SP-3 |

## Terms PRIMER Avoids

Deliberately absent from client-facing output: BPMN notation, Six Sigma
belt vocabulary, and vendor or product names. The methodology is written for
adoption by business teams; jargon that requires certification to parse
defeats the purpose, and vendor names compromise the evidence posture.
